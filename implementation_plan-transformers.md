# Adding Transformer Architecture Support

Expanding Blink to natively support Transformer architectures (specifically Vision Transformers like ViT, Swin, DeiT) requires updates across the entire pipeline—from data collection and static feature parsing down to the SHAP explainability dashboard.

## Proposed Changes

We will systematically update the codebase to extract transformer-specific features, profile them, and retrain the predictive ecosystem.

### 1. Advanced Feature Extraction
The GNN already maps `MultiheadAttention` and `LayerNorm` nodes, but the static tabular features need to be transformer-aware.

#### [MODIFY] [model_analyser.py](file:///c:/Aniket/review%20blink/Neusight/model_analyser.py)
- Add extraction for `max_embed_dim` (the width of the transformer tokens).
- Ensure `num_attn_layers` and `num_layernorm_layers` are explicitly surfaced to the XGBoost predictor.
- Update the mock-activation memory estimation to explicitly account for [(seq_len) x (embed_dim)](file:///c:/Aniket/review%20blink/Neusight/prediction_model.py#343-378) tensor sizes generated by Vision Transformers, rather than just [(H x W x C)](file:///c:/Aniket/review%20blink/Neusight/prediction_model.py#343-378) convolutions.

### 2. Dataset Expansion
We need ground-truth timing and memory consumption for PyTorch's native transformer models.

#### [MODIFY] [scripts/collect_data.py](file:///c:/Aniket/review%20blink/Neusight/scripts/collect_data.py)

### 3. Training the Unified Ecosystem
Once the new data is generated, we must retrain the predictors so they can generalize to the new architecture family.
